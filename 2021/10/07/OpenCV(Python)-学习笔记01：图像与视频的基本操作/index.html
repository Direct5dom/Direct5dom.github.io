下载安装

这里有几个选择：

  • opencv-python：只包含opencv库的主要模块. 一般不推荐安装.
  • opencv-contrib-python：包含主要模块和contrib模块, 功能基本完整, 推荐安装.
  • opencv-python-headless：和opencv-python一样, 但是没有GUI功能, 无外设系统可用.
  • opencv-contrib-python-headless：和opencv-contrib-python一样但是没有GUI功能. 无外设系统可用.

因此一般来说都会选择安装opencv-contrib-python

不要同时安装opencv-python和opencv-contrib-python

图像入门

读取图像（cv.imread()）

使用cv.imread()函数读取一张图像，图片应该在工作目录中，或者应该提供完整的图像路径。

来自：OpenCV: Image file reading and writing

其中参数flags指定了读取的方式：

  • cv.IMREAD_COLOR：加载彩色图像，任何图像的透明度都会被忽略，它是默认flag
  • cv.IMREAD_GRAYSCALE：以灰度模式加载图像
  • cv.IMREAD_UNCHANGED：加载图像，包括 alpha 通道

NOTE：可以简单的通过1、0、-1来调用这三个flag，而不是输入flag全名。

显示图像（cv.imshow()）

用cv.imshow()函数在窗口中显示图像，窗口自动适应图像的大小。

来自：OpenCV: Image file reading and writing

第一个参数是窗口名，它是一个字符串，第二个参数就是我们的图像。你可以根据需要创建任意数量的窗口，但是窗口名字要不同。

如果单独用cv.imshow()，会发现打开的窗口一瞬间被关闭，一方面这是因为Python程序运行终止导致的；另一方面在OpenCV中还需要另一个函数去会处理许多其他GUI事件才能保证窗口的持续存在。

等待键盘函数（cv.waitKey()）

cv.waitKey()是一个键盘绑定函数，它的参数是以毫秒为单位的时间。该函数为任意键盘事件等待指定毫秒。如果你在这段时间内按下任意键，程序将继续。如果传的是0，它会一直等待键盘按下。

它也可以设置检测特定的击键。

视频入门

从相机捕捉视频

OpenCV提供了一个简单的接口，用于捕捉视频内容。

VideoCapture的对象

要捕获视频，需要创建一个**VideoCapture**的对象。

它的可以有两个参数：

  • filename：一个视频文件名。
  • index：一个设备索引。

设备索引仅仅是摄像机编号，从0开始计算。之后可以逐帧捕获。但是最后，不要忘记释放这个VideoCapture对象。

捕获帧（cap.read()）

我们使用cap.read()来进行帧捕获。而要进行视频的捕获就必须使用循环。

cap.read()返回一个bool值(True/False)。如果加载成功，它会返回True。因此，你可以通过这个返回值判断视频是否结束。

有时，cap可能没有初始化capture。在这种情况下，此代码显示错误。你可以通过cap.isOpened()检查它是否初始化。如果它是True，那么意味着cap成功初始化；否则意味着未成功初始化，需要用cap.open()打开在使用。

获取一些视频的特征（cv.VideoCapture.get( propID )）

可以通过使用**cap.get(propId)函数获取一些视频的特征。或者使用cv.VideoCapture.get()获取全部细节。它们中有些值可以使用cap.set(propId, value)**修改。Value就是你想要的新值

这里的 propld 是一个 0-18 的数字，每个数字代表视频的一个特征（如果这个视频有）。

例如：

  • 使用cap.get(cv.CAP_PROP_FRAME_WIDTH)获得宽，

  • 使用cap.get(cv.CAP_PROP_FRAME_HEIGHT) 获得高。

它返回的是 640x480，但是我想把它修改为 320x240。

  • 使用ret = cap.set(cv.CAP_PROP_FRAME_WIDTH,320)和ret = cap.set(cv.CAP_PROP_FRAME_HEIGHT,240)

NOTE：如果报错，确保相机用任意其他的相机程序 (如 Linux 下的 Cheese 程序) 可以正常工作。

保存视频

在图像上，我们使用**cv.imwrite()**去保存，这里需要更多的工作。

这次需要创建一个**VideoWriter**对象。我们应该指定输出文件的名字 (例如：output.avi)。

之后我们应该指定FourCC码。再之后传递每秒帧数和帧大小。最后一个是isColor flag。如果是True，编码器期望彩色帧，否则它适用于灰度帧。

FourCC是用于指定视频解码器的4字节代码。fourcc.org是可用编码的列表。它取决于平台，下面的编码就很好：

  • In Fedora: DIVX, XVID, MJPG, X264, WMV1, WMV2. (XVID 是最合适的. MJPG 结果比较大. X264 结果比较小)
  • In Windows: DIVX (还需要测试和添加跟多内容)
  • In OSX: MJPG (.mp4), DIVX (.avi), X264 (.mkv).

*对于MJPG，FourCC的代码作为cv.VideoWriter_fourcc('M','J','P','G')或 cv.VideoWriter_fourcc('MJPG') 传递。
