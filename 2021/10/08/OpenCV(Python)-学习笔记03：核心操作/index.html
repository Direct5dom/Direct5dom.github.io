图像加法（cv.add()）

OpenCV提供了一个简单的函数cv.add()来对两个图像进行加法操作。或者也可以通过Numpy操作res = img1 + img2。两个图像应该具有相同的深度和类型，或者第二个图像可以只是一个标量值。

OpenCV加法和Numpy加法是有区别的。OpenCV加法是饱和运算，Numpy加法是模运算。两种加法对于结果溢出的数据，会通过某种方法使其在限定的数据范围内。

图像混合（cv.addWeighted()）

图像混合也是将两个图像进行叠加，只不过和上面所说的图像加法不同。混合会赋予图像不同的权重，从而混合出透明感。用数学语言表达则是：
$$
g(x)=(1-a)f_{0}(x)+af_{1}(x)
$$
从 0 → 1 改变 α ，相当于从一个图像到另一个图像的过渡。

比如我们混合两个图像，第一幅的权重为 0.7，第二幅则为 0.3，使用函数cv.addWeighted()对图像应用如下公式:
$$
dst=\alpha \cdot img1+\beta \cdot img2+\gamma
$$

这里取γ为零。

在进行图像处理时，由于每秒需要处理大量操作，因此我们的代码不仅要正确，还要更快。

OpenCV中的默认优化

许多OpenCV函数都使用了SSE2、AVX等进行了优化，在使用时是默认启用的。

可以使用cv.useOptimized()来检查是否启用，并使用cv.setUseOptimized()来执行启用/禁用的操作。

如您所见，优化的中值过滤比未优化的版本快 2 倍。 如果您查看其来源，您会发现中值滤波是经过 SIMD 优化的。 因此，您可以使用它在代码顶部启用优化（请记住，默认情况下启用它）。

性能优化技术

有几种技术和编码方法可以利用 Python 和 Numpy 的最大性能。这里仅注明相关的内容，并提供重要来源的链接。这里要注意的主要是，首先尝试以简单的方式实现算法。一旦它正常工作，对其进行分析，找到瓶颈并对其进行优化。

  1. 尽量避免在 Python 中使用循环，尤其是双循环/三循环等。它们本质上很慢。
  2. 将算法/代码矢量化到最大可能范围，因为 Numpy 和 OpenCV 针对向量运算进行了优化。
  3. 利用缓存一致性。
  4. 除非需要，否则永远不要复制数组。尝试使用视图。复制多维数组是一项耗费时间和空间都很多的操作。(译者注：译者认为这儿的意思是除非必要，否在尽量不要使用深拷贝)

即使在完成所有这些操作之后，如果您的代码仍然很慢，或者使用大型循环是不可避免的，请使用其他库（如 Cython）来加快速度。
